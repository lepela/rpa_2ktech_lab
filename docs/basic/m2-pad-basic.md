## 1. 화면 구성 파악

### 콘솔(Console)
PAD를 처음 열면 보이는 화면입니다.

- **내 흐름** 목록 관리
- 흐름 실행/편집/삭제
- 설정 및 계정 관리

<!-- SCREENSHOT: PAD 콘솔 메인 화면 — 흐름 목록이 보이는 상태 -->

### 디자이너(Designer)
흐름을 편집하는 화면입니다. 흐름을 더블클릭하면 열립니다.

```
┌─────────────────────────────────────────────┐
│  [액션 패널]  │    [캔버스]    │  [변수 패널] │
│              │                │              │
│  검색         │  1. 액션 1     │  입력 변수   │
│  > 브라우저   │  2. 액션 2     │  출력 변수   │
│  > 엑셀       │  3. 액션 3     │  흐름 변수   │
│  > 메시지     │  ...           │              │
└─────────────────────────────────────────────┘
```

| 영역 | 역할 |
|------|------|
| **액션 패널** | 사용 가능한 액션 목록 (드래그해서 캔버스에 추가) |
| **캔버스** | 흐름 순서를 설계하는 공간 |
| **변수 패널** | 흐름에서 사용 중인 변수 목록 및 실시간 값 확인 |

## 2. 변수

### 변수란?

변수는 **"이름표 붙은 상자"**입니다.  
어떤 값을 임시로 담아두는 공간이라고 생각하세요.

예를 들어, `%오늘날짜%`라는 변수에 "2024-01-15"를 담아두면, 나중에 `%오늘날짜%`라고 쓸 때마다 "2024-01-15"가 나옵니다.

### 변수 타입

| 타입 | 한 줄 설명 | 예시 |
|------|-----------|------|
| **텍스트** | 글자 상자 | `"서울"`, `"홍길동"` |
| **숫자** | 숫자 상자 | `2024`, `3.14` |
| **Boolean** | 켜짐/꺼짐 | `True`, `False` |
| **리스트** | 같은 종류 묶음 | `["서울", "부산", "대구"]` |
| **데이터테이블** | 엑셀 표 그 자체 | For Each 실습에서 등장 |

{: .note }
> **Object 타입**은 오늘 범위를 넘어서니 생략합니다. 나중에 필요하면 그때 배워도 충분합니다.

### 변수 참조 방법

PAD에서 변수를 쓸 때는 변수명 앞뒤에 `%`를 붙입니다.

```
%CurrentDateTime%   → 현재 날짜/시간
%ExcelData%         → 엑셀에서 읽어온 데이터
%UserName%          → 내가 만든 변수
```

{: .note }
> **팁:** 변수 패널에서 흐름 실행 중 변수 값이 실시간으로 변하는 것을 눈으로 확인할 수 있습니다. 디버깅의 핵심입니다.

## 3. 제어구조

### If / Else — 조건에 따라 다르게

```
If [조건] Then
    액션 A 실행
Else
    액션 B 실행
End If
```

**예시:** 파일이 있으면 열고, 없으면 새로 만들기

### For Each — 목록을 하나씩 처리

```
For Each [항목] in [목록]
    해당 항목으로 액션 실행
End For
```

**예시:** 엑셀에서 읽어온 100행을 한 줄씩 처리하기

### Loop — N번 반복

```
Loop [N]번
    액션 반복 실행
End Loop
```

**예시:** 같은 버튼을 5번 클릭하기

### Switch — 복잡한 조건 분기

여러 가지 경우(Case)에 따라 다른 액션을 실행합니다.  
오늘은 언급만 하고, 필요할 때 사용하면 됩니다.

## 4. 흐름제어 — 서브흐름(하위 흐름)

### 서브흐름이란?

큰 흐름을 역할별로 나누는 방법입니다.  
마치 긴 문서를 장(Chapter)으로 나누는 것처럼, 흐름도 기능별로 나눌 수 있습니다.

```
Main (전체 흐름 지휘)
  ├── GetData (데이터 수집)
  ├── ProcessData (데이터 처리)
  └── SendReport (결과 발송)
```

**장점:**
- 흐름이 복잡해져도 이해하기 쉽습니다.
- 문제가 생기면 어느 서브흐름인지 바로 찾을 수 있습니다.
- 서브흐름을 재사용할 수 있습니다.

## 5. Input / Output Parameter

PA(클라우드 흐름)에서 PAD(데스크톱 흐름)를 호출할 때 데이터를 주고받는 방법입니다.

```
PA(클라우드)  →  [Input 파라미터 전달]  →  PAD(데스크톱)
PA(클라우드)  ←  [Output 파라미터 반환]  ←  PAD(데스크톱)
```

**예시:**
- Input: PA에서 "처리할 파일 경로"를 PAD에 전달
- Output: PAD가 처리 완료 후 "처리 건수"를 PA로 반환

{: .note }
> 오늘 실습에서는 직접 사용하지 않습니다. M5에서 PA-PAD 연결을 다룰 때 다시 등장합니다.

## ✅ 핵심 정리

- PAD 화면 = 콘솔(목록 관리) + 디자이너(흐름 설계)
- 변수 = 이름표 붙은 상자, `%변수명%`으로 참조
- 변수 패널에서 실행 중 값을 실시간 확인 가능
- 제어구조: If(조건 분기), For Each(목록 반복), Loop(횟수 반복)
- 서브흐름으로 큰 흐름을 역할별로 분리
