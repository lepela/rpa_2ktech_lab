# M5. PA 클라우드 흐름 맛보기

## 🎯 학습 목표

- PA 클라우드 흐름의 3가지 종류를 이해합니다.
- 커넥터의 개념을 파악합니다.
- PAD와 PA가 어떻게 연결되는지 전체 그림을 봅니다.
- 템플릿으로 간단한 자동화 흐름을 체험합니다.

| 구분 | 시간 |
|------|------|
| 강사 설명 | 10분 |

## 1. PA 클라우드 흐름 3종

`make.powerautomate.com`에 접속하면 3가지 흐름을 만들 수 있습니다.

| 종류 | 트리거 | 예시 |
|------|--------|------|
| **인스턴트(즉시)** | 버튼 클릭 | 버튼 누르면 즉시 실행 |
| **자동화** | 특정 이벤트 발생 | 이메일 도착 시 자동 실행 |
| **예약** | 정해진 시간 | 매일 오전 9시에 실행 |

---

## 2. 커넥터 개념

커넥터는 **PA와 외부 서비스를 연결하는 다리**입니다.

```
[PA 흐름]
   ↕ Outlook 커넥터
[Microsoft Outlook]

   ↕ Teams 커넥터
[Microsoft Teams]

   ↕ SharePoint 커넥터
[SharePoint]
```

### 표준 vs 프리미엄

| 구분 | 설명 | 라이선스 |
|------|------|--------|
| **표준 커넥터** | Outlook, Teams, SharePoint 등 | 기본 포함 |
| **프리미엄 커넥터** | Salesforce, ServiceNow 등 | 별도 라이선스 필요 |

{: .warning }
> 오늘 실습에서는 표준 커넥터만 사용합니다. 프리미엄 커넥터는 라이선스 확인 후 사용하세요.

---

## 3. PAD와 PA는 이렇게 연결됩니다

전체 자동화 구조를 한눈에 보면:

```
[트리거] 이메일 도착
    ↓
[PA 클라우드 흐름]
  - 이메일에서 데이터 추출
  - PAD 흐름 호출 (Input 파라미터 전달)
    ↓
[PAD 데스크톱 흐름]
  - 내 PC에서 앱 자동화
  - 처리 완료 후 결과 반환 (Output 파라미터)
    ↓
[PA 클라우드 흐름 계속]
  - Teams로 완료 알림 발송
```

{: .note }
> **핵심 메시지:**  
> "PAD에서 `%변수%`로 값을 넣은 것처럼  
>  PA에서도 동적 콘텐츠로 값을 넣습니다.  
>  같은 개념, 다른 모양입니다."

## HOL 실습

### HOL 1. 이메일 → Teams 채널 전달 (10분)

1. `make.powerautomate.com` 접속
2. **[만들기] > [템플릿으로 시작]** 클릭
3. 검색창에 `Email to Teams` 입력
4. **"새 이메일 수신 시 Teams 채널에 게시"** 템플릿 선택
5. 커넥터 연결 확인 (Outlook, Teams)
6. **[계속]** → 채널 선택 → **[흐름 만들기]**
7. 테스트 이메일 발송 → Teams 채널 확인

<!-- SCREENSHOT: PA 템플릿 선택 화면 — Email to Teams 템플릿 강조 -->

### HOL 2. 예약 흐름 — 자동 메일 발송 (10분)

1. **[만들기] > [예약된 클라우드 흐름]** 클릭
2. 설정:
   - **흐름 이름:** `매일아침_인사메일`
   - **반복:** 1일 / 오전 9시
3. **[만들기]** → 디자이너 열림
4. **[+ 새 단계]** → `이메일 보내기` 검색 → **Outlook 이메일 보내기** 선택
5. 받는 사람: 본인 이메일 / 제목: `좋은 아침입니다!`
6. **[저장]** → **[테스트]** (수동 실행)

## ✅ 핵심 정리

- PA 클라우드 흐름 = 인스턴트 / 자동화 / 예약 3종
- 커넥터 = PA와 외부 서비스의 연결 다리
- PA ↔ PAD 연결로 클라우드 + 데스크톱 자동화를 결합 가능
- `%변수%`(PAD)와 동적 콘텐츠(PA)는 같은 개념
