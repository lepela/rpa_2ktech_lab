# M3 실습 — 오피넷 유가 → 엑셀 기록

## 🎯 학습 목표

- 웹 데이터 추출 → 엑셀 기록의 전체 흐름을 완주합니다.
- 서브흐름 구조로 흐름을 역할별로 분리하는 방법을 익힙니다.
- 안정적인 CSS 셀렉터를 직접 만들어봅니다.

## ⏱ 예상 소요 시간

| 구분 | 시간 |
|------|------|
| 실습 | 50분 |
| 확장 실습 | 10분 |

---

## 준비물

- Chrome + PAD 확장 설치 완료
- 오피넷 접속 가능한 네트워크 환경

---

## 시나리오

오피넷(`www.opinet.co.kr`)에서 현재 휘발유 평균 가격을 조회하고,  
날짜와 함께 엑셀 파일에 누적 기록합니다.

---

## 서브흐름 구조

```
Main
  ├── GetFuelPrice   (오피넷 스크래핑)
  └── WriteToExcel   (엑셀 기록)
```

---

## 단계별 가이드

### 1단계. 새 흐름 생성 + 서브흐름 추가

1. PAD 콘솔 → **[+ 새 흐름]** → 이름: `M3_오피넷_유가기록`
2. 디자이너 열리면 하단 **[+]** 버튼 → **[서브흐름 추가]**
3. 이름: `GetFuelPrice` → 확인
4. 같은 방법으로 `WriteToExcel` 서브흐름 추가

### 2단계. GetFuelPrice 서브흐름 작성

**GetFuelPrice 탭 클릭 후 시작합니다.**

#### 2-1. Chrome 시작
1. 액션 패널에서 **[브라우저 자동화] > [새 Chrome 시작]** 추가
2. 설정:
   - **초기 URL:** `https://www.opinet.co.kr`
   - **저장 변수:** `Browser` (기본값 유지)
3. 저장

#### 2-2. 자동캡처 시도 (설계된 실패)
1. **[웹 페이지에서 데이터 추출]** 액션 추가
2. **[UI 요소 선택]** 클릭 후 오피넷 유가 숫자 클릭
3. 자동 생성된 셀렉터 확인 (구조 경로 방식임을 확인)
4. **저장 없이 취소** 또는 저장 후 실행 → 오류 확인

<!-- SCREENSHOT: 자동캡처 후 생성된 셀렉터 예시 -->

#### 2-3. CSS 셀렉터로 교체

1. Chrome에서 `Ctrl+Shift+C` → 오피넷 유가 숫자 클릭
2. Elements 탭에서 해당 요소 확인
3. 아래 안정적인 셀렉터 사용:

```css
#oilcon1 .oil_info dl:first-child dd span.text-3
```

4. **[웹 페이지의 UI 요소에서 데이터 추출]** 액션에서 셀렉터를 위 값으로 직접 입력
5. **저장 변수:** `FuelPrice`
6. 저장

<!-- SCREENSHOT: 오피넷 유가 요소 선택 화면 — Elements 탭에서 해당 span 요소 강조 -->

---

### 3단계. WriteToExcel 서브흐름 작성

**WriteToExcel 탭 클릭 후 시작합니다.**

#### 3-1. Excel 시작 (새 파일)
1. **[Excel] > [Excel 시작]** 액션 추가
2. 설정:
   - **Excel 시작:** 빈 문서 (새 파일)
   - **저장 변수:** `ExcelInstance`
3. 저장

#### 3-2. 현재 날짜 가져오기
1. **[날짜 및 시간] > [현재 날짜 및 시간 가져오기]** 액션 추가
2. **저장 변수:** `CurrentDateTime`
3. 저장

#### 3-3. 엑셀에 데이터 쓰기
1. **[Excel] > [워크시트에 쓰기]** 액션 추가 — 날짜 기록
2. 설정:
   - **Excel 인스턴스:** `%ExcelInstance%`
   - **쓸 값:** `%CurrentDateTime%`
   - **열:** `1` / **행:** `1`
3. 저장

4. **[워크시트에 쓰기]** 액션 하나 더 추가 — 유가 기록
5. 설정:
   - **쓸 값:** `%FuelPrice%`
   - **열:** `2` / **행:** `1`
6. 저장

{: .note }
> 매일 실행해서 누적 기록하려면 행 번호를 변수로 관리해야 합니다.  
> 오늘은 1행에 기록하는 것으로 시작합니다. 확장 방법은 LLM에게 물어보세요.

#### 3-4. Excel 저장 후 종료
1. **[Excel] > [Excel 저장]** 액션 추가
2. 설정:
   - **저장 모드:** 다른 이름으로 저장
   - **파일 경로:** `C:\Users\[사용자명]\Desktop\유가기록.xlsx`
3. 저장

4. **[Excel] > [Excel 닫기]** 액션 추가
5. **Excel 인스턴스:** `%ExcelInstance%`
6. 저장

### 4단계. Main에서 두 서브흐름 호출

**Main 탭 클릭 후 시작합니다.**

1. **[흐름 제어] > [서브흐름 실행]** 액션 추가
   - **서브흐름:** `GetFuelPrice`
2. **[서브흐름 실행]** 액션 하나 더 추가
   - **서브흐름:** `WriteToExcel`

완성된 Main 흐름:
```
1. 서브흐름 실행 → GetFuelPrice
2. 서브흐름 실행 → WriteToExcel
```

---

### 5단계. 실행 및 확인

1. **[▶ 실행]** 클릭
2. Chrome이 열리고 오피넷으로 이동하는 것 확인
3. 흐름 완료 후 `유가기록.xlsx` 파일 열기
4. A1셀에 날짜, B1셀에 유가가 기록되었는지 확인

## 확장 실습 — 직접 해보기

같은 패턴으로 지역 평균 가격도 추가해보세요.

**지역 평균 가격 셀렉터:**
```css
#sido_price1 span.text-3
```

1. GetFuelPrice 서브흐름에 데이터 추출 액션 하나 더 추가
2. 위 셀렉터 사용, 저장 변수: `RegionalPrice`
3. WriteToExcel에서 C1셀에 `%RegionalPrice%` 기록

---

## 💡 LLM 활용 팁

셀렉터를 직접 찾고 싶다면:

1. `F12` → Elements 탭에서 원하는 요소 찾기
2. 우클릭 → **Copy** → **Copy selector**
3. LLM에게:

```
이 CSS 셀렉터를 PAD 웹자동화에 최적화해줘:
[복사한 셀렉터 붙여넣기]
```

## ✅ 핵심 정리

- 서브흐름 구조로 역할을 분리하면 복잡한 흐름도 관리하기 쉽습니다.
- 자동캡처 셀렉터는 불안정하고, id/class 기반 셀렉터가 안정적입니다.
- 웹 추출 → 엑셀 기록의 전체 흐름을 완주했습니다.
