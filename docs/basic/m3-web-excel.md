# M3. 웹+엑셀 자동화

## 🎯 학습 목표

- PAD의 브라우저 자동화 방식을 이해합니다.
- 셀렉터(Selector) 개념을 파악하고 안정적인 셀렉터 정제 방법을 익힙니다.
- PAD의 엑셀 자동화 액션(실행/읽기/쓰기/저장)을 이해합니다.

## 1. 브라우저 자동화 개요

PAD는 Chrome과 Edge를 자동화할 수 있습니다.

### 준비 확인

- Chrome 설치 완료
- Chrome용 PAD 확장 프로그램 설치 완료

{: .warning }
> Chrome 확장이 설치되지 않으면 웹 자동화가 작동하지 않습니다.  
> PAD 콘솔 → **도구** → **브라우저 확장** → **Chrome** 에서 설치할 수 있습니다.

## 2. 셀렉터 — 핵심 개념

### HTML이 뭔지 몰라도 됩니다

이것만 알면 됩니다:

> 브라우저의 모든 요소(버튼, 텍스트, 입력창)에는 **"주소"**가 있습니다.
>
> - **id:** 페이지에서 단 하나뿐인 고유 주소
> - **class:** 비슷한 요소들이 공유하는 그룹 주소

PAD가 웹 요소를 찾을 때 이 주소(셀렉터)를 사용합니다.

### 셀렉터 예시

```css
#oilcon1 .oil_info dd span.text-3
```

이 셀렉터의 의미:
- `#oilcon1` → id가 "oilcon1"인 요소 안에서
- `.oil_info` → class가 "oil_info"인 요소 안에서
- `dd span.text-3` → dd 태그 안의 class가 "text-3"인 span 태그

---

## 3. PAD 자동캡처의 한계

### 자동캡처란?

PAD에서 "UI 요소 캡처" 버튼을 누르고 원하는 요소를 클릭하면 자동으로 셀렉터를 만들어줍니다.

### 왜 자동캡처가 불안정한가요?

자동 생성된 셀렉터는 종종 이렇게 생겼습니다:

```
:nth-child(3) > :nth-child(2) > span
```

이 셀렉터는 "3번째 자식의 2번째 자식의 span"을 의미합니다.  
페이지 구조가 조금이라도 바뀌면 **즉시 깨집니다**.

### 설계된 실패 체험

실습 중 자동캡처 그대로 실행해서 오류가 나는 것을 직접 확인합니다.  
오류는 실패가 아닙니다. "왜 안정적인 셀렉터가 필요한지" 이해하는 과정입니다.

---

## 4. 셀렉터 정제 방법

아래 6단계를 따라하면 안정적인 셀렉터를 만들 수 있습니다.

```
① PAD 자동캡처로 일단 요소 잡기
② 실행 → 깨지는 것 확인 (설계된 실패)
③ Chrome에서 Ctrl+Shift+C → 원하는 요소 클릭
④ Elements 탭에서 코드 확인 후 우클릭 → Copy → Copy selector
⑤ LLM에 붙여넣기:
   "이 코드에서 PAD 웹자동화에 쓸 최적의 CSS 셀렉터 찾아줘: [붙여넣기]"
⑥ 나온 셀렉터를 PAD에 직접 입력 → 테스트
```

<!-- SCREENSHOT: Chrome Ctrl+Shift+C 인스펙터 화면 — Elements 탭, Copy selector 위치 -->

{: .note }
> **셀렉터 안정성 순서:** id > 의미있는 class > 구조 경로  
> id가 있으면 무조건 id를 씁니다. `#oilcon1`처럼요.

## 5. 엑셀 자동화 개요

### 주요 액션

| 액션 | 역할 |
|------|------|
| **Excel 시작** | 새 파일 생성 또는 기존 파일 열기 |
| **워크시트에서 읽기** | 셀/범위/전체 데이터 읽어오기 |
| **워크시트에 쓰기** | 특정 셀에 값 입력 |
| **Excel 저장** | 파일 저장 |
| **Excel 닫기** | Excel 인스턴스 종료 |

### 데이터 읽기 방식

| 방식 | 설명 | 결과 변수 타입 |
|------|------|--------------|
| 단일 셀 값 | 특정 셀 하나의 값 | 텍스트/숫자 |
| 셀 범위 | 지정한 범위 읽기 | 데이터테이블 |
| 전체 읽기 | 데이터가 있는 전체 범위 | 데이터테이블 |

## 💡 LLM 활용 팁

셀렉터 문제를 LLM으로 해결하는 방법:

1. `F12` → Elements 탭에서 원하는 요소 찾기
2. 우클릭 → **Copy** → **Copy selector**
3. LLM에게:

```
이 CSS 셀렉터를 PAD 웹자동화에 최적화해줘.
id나 의미있는 class를 우선 사용하도록 단순화해줘.

[여기에 복사한 셀렉터 붙여넣기]
```

## ✅ 핵심 정리

- 웹 자동화 = 셀렉터로 요소를 찾아서 액션 실행
- 자동캡처 셀렉터는 불안정 → Ctrl+Shift+C + LLM으로 정제
- id 기반 셀렉터가 가장 안정적
- 엑셀 = 시작 → 읽기/쓰기 → 저장 → 닫기 순서
