# M4. UI+이메일 자동화

## 🎯 학습 목표

- 데스크톱 앱의 UI 요소를 자동화하는 방법을 이해합니다.
- 마우스/키보드 액션으로 앱을 제어하는 방법을 익힙니다.
- PAD에서 Outlook을 통해 이메일을 발송하는 방법을 배웁니다.

## ⏱ 예상 소요 시간

| 구분 | 시간 |
|------|------|
| 강사 설명 | 20분 |
| 실습 | 70분 |
| 버퍼 | 10분 |

---

## 1. UI 자동화 개요

### 브라우저 자동화와의 공통점

M3에서 배운 웹 자동화와 구조가 동일합니다.

| 항목 | 웹 자동화 | UI 자동화 |
|------|----------|----------|
| 대상 | 브라우저 안의 요소 | 데스크톱 앱의 요소 |
| 셀렉터 방식 | CSS 셀렉터 | UI 선택기(Selector) |
| 캡처 방법 | Ctrl+Shift+C | Ctrl+클릭 (PAD 캡처 모드) |

### UI 요소 캡처 방법

1. **[UI 자동화] > 관련 액션** 추가
2. **[UI 요소 선택]** 버튼 클릭
3. PAD가 캡처 모드로 전환됨 (화면 테두리 빨간색)
4. 원하는 앱 요소 위에 마우스를 올리면 강조 표시
5. **Ctrl+클릭** → 해당 요소 캡처

<!-- SCREENSHOT: PAD 캡처 모드 — 앱 요소가 빨간 테두리로 강조된 화면 -->

### 선택기 직접 수정

자동캡처 후 선택기가 너무 구체적으로 잡히면 직접 수정합니다.

1. 액션 편집 창에서 **[UI 요소]** 옆 **[편집]** 클릭
2. 선택기 편집기에서 불필요한 속성 제거
3. 모르겠으면 LLM에게:

```
이 PAD UI 선택기에서 불필요한 속성을 제거해서 단순화해줘:
[선택기 붙여넣기]
```

---

## 2. 마우스/키보드 액션

| 액션 | 용도 |
|------|------|
| **창에서 UI 요소 클릭** | 버튼, 링크 클릭 |
| **창에서 텍스트 필드 채우기** | 입력창에 텍스트 입력 |
| **드롭다운 목록에서 선택** | 드롭다운 선택 |
| **창에서 UI 요소의 세부 정보 가져오기** | 요소 값/속성 읽기 |

{: .note }
> **`%변수명%` 활용:** "텍스트 필드 채우기" 액션의 텍스트 값에 `%변수명%`을 쓰면 변수 값이 자동으로 입력됩니다.

---

## 3. PAD Outlook 액션

### 연결 참조 확인

PAD에서 Outlook 이메일을 보내려면 Outlook이 설치되어 있고 계정이 연결되어 있어야 합니다.

1. PAD 디자이너에서 **[이메일] > [Outlook을 통해 이메일 메시지 보내기]** 액션 확인
2. 또는 **[이메일] > [이메일 메시지 보내기]** (SMTP 방식)

### 이메일 보내기 액션 설정

| 설정 항목 | 예시 값 |
|----------|--------|
| **받는 사람** | `recipient@company.com` |
| **제목** | `자동화 완료 알림` |
| **본문** | `처리가 완료되었습니다. 처리 건수: %처리건수%건` |

### 동적 콘텐츠로 변수 삽입

본문에 `%변수명%`을 쓰면 실행 시 변수 값으로 대체됩니다.

```
본문 예시:
안녕하세요.

%처리날짜% 기준으로 총 %처리건수%건의 데이터 입력이 완료되었습니다.

감사합니다.
```

{: .note }
> 이 개념은 M5에서 배울 PA(클라우드 흐름)의 **동적 콘텐츠**와 완전히 같습니다.  
> PAD에서는 `%변수명%`, PA에서는 동적 콘텐츠 선택기 — 모양은 다르지만 원리가 같습니다.

## 💡 LLM 활용 팁

UI 요소를 찾지 못할 때:

```
PAD UI 자동화에서 앱 버튼을 찾지 못할 때 
선택기를 어떻게 수정하면 좋을지 알려줘.
현재 선택기: [선택기 붙여넣기]
```

---

## ✅ 핵심 정리

- UI 자동화 = 웹 자동화와 같은 구조, 대상이 데스크톱 앱으로 바뀐 것
- Ctrl+클릭으로 UI 요소 캡처, 선택기 너무 구체적이면 직접 수정
- 이메일 본문에 `%변수명%`으로 동적 값 삽입 가능
