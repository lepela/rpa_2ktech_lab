# T4. 트러블슈팅 레퍼런스

자주 발생하는 오류와 빠른 해결 방법을 정리합니다.

---

## PAD 설치 오류

### 증상
- PAD 설치 중 오류 메시지
- "이 앱은 사용자 PC에 설치할 수 없습니다"

### 원인 & 해결

| 원인 | 해결 방법 |
|------|----------|
| 조직 정책으로 설치 차단 | IT 부서에 PAD 설치 허용 요청 |
| Windows Home — 일부 기능 제한 | 오늘 실습(Attended)은 지원됨, 진행 가능 |
| .NET 버전 미충족 | Windows Update 실행 후 재시도 |
| 이전 버전 충돌 | 제어판 > 앱 > PAD 제거 후 재설치 |

{: .note }
> Microsoft Store에서 "Power Automate"로 검색하면 최신 버전으로 설치할 수 있습니다.

## 브라우저 확장 연결 안 됨

### 증상
- PAD에서 "브라우저를 시작할 수 없습니다" 오류
- Chrome 자동화 액션이 작동하지 않음

### 해결 방법

1. Chrome 확장 설치 확인
   - Chrome 주소창: `chrome://extensions/`
   - "Microsoft Power Automate" 확장 찾기
   - 토글 **켜짐** 상태인지 확인

2. 확장이 없으면 설치
   - PAD 콘솔 → **[도구]** → **[브라우저 확장]** → Chrome 설치 클릭

3. Chrome 완전 종료 후 재시작
   - 트레이에서 Chrome 완전 종료 → 재실행

4. PAD 재시작 후 다시 시도

## 셀렉터로 요소를 찾을 수 없음

### 증상
- "UI 요소를 찾을 수 없습니다" 오류
- 흐름이 엉뚱한 요소를 클릭함

### 해결 방법

**Step 1. 셀렉터 다시 캡처**
1. 해당 액션 편집 → **[UI 요소]** 다시 캡처
2. Ctrl+클릭으로 정확한 요소 선택

**Step 2. CSS 셀렉터로 교체**
```
① Chrome에서 Ctrl+Shift+C
② 원하는 요소 클릭
③ Elements 탭 > 우클릭 > Copy > Copy selector
④ LLM으로 최적화 후 PAD에 직접 입력
```

**Step 3. 대기 추가 (동적 로딩 페이지)**
- **[웹 자동화] > [웹 페이지의 UI 요소 대기]** 액션 추가
- 요소가 나타날 때까지 기다린 후 클릭

**Step 4. 페이지 로드 대기**
- **[웹 자동화] > [웹 페이지 로드 기다리기]** 액션 추가

## Excel 파일 잠김 오류

### 증상
- "파일이 다른 프로세스에서 사용 중입니다"
- Excel 액션에서 파일 열기 실패

### 해결 방법

1. 이미 열려있는 Excel 인스턴스 모두 닫기
2. 작업 관리자(`Ctrl+Shift+Esc`) → Excel 프로세스 강제 종료
3. 파일 경로에 한글 또는 특수문자 있으면 영문 경로로 변경
4. OneDrive 동기화 중이면 동기화 완료 후 실행

{: .warning }
> 흐름이 중간에 중단되면 Excel 인스턴스가 열린 채로 남을 수 있습니다.  
> **[Excel] > [Excel 닫기]** 액션을 항상 마지막에 추가하세요.

## Outlook 커넥터 인증 오류

### 증상 (PA 클라우드 흐름)
- "연결 오류" 또는 "인증이 필요합니다"
- 이메일 발송 액션 실패

### 해결 방법

1. `make.powerautomate.com` → **[데이터]** → **[연결]**
2. Outlook 연결 상태 확인 → 오류 표시 있으면 **[수정]** 클릭
3. Microsoft 365 계정으로 다시 인증
4. 흐름으로 돌아가서 이메일 액션 저장

### 증상 (PAD)
- "Outlook 계정을 찾을 수 없습니다"

### 해결 방법
1. PC에 Outlook 앱 설치 확인
2. Outlook 열어서 계정 로그인 상태 확인
3. 계정 로그인 후 PAD 재실행

## 흐름 실행 중 예상치 못한 종료

### 증상
- 흐름이 중간에 갑자기 멈춤
- 오류 메시지 없이 종료됨

### 해결 방법

**원인 파악:**
1. 디자이너 → **[흐름 실행 기록]** 확인 (우측 패널)
2. 어느 액션에서 멈췄는지 확인

**자주 있는 원인:**

| 원인 | 확인 방법 | 해결 |
|------|----------|------|
| 팝업창 처리 안 됨 | 실행 중 화면 관찰 | 팝업 닫기 액션 추가 |
| 변수값이 비어있음 | 변수 패널에서 값 확인 | 이전 단계 검토 |
| 시간 초과 | 느린 페이지 로딩 | 대기 액션 추가 |
| 화면 해상도 변경 | UI 요소 위치 변경 | 요소 다시 캡처 |

**디버깅 팁:**
- **[메시지 표시]** 액션으로 중간 변수 값 확인
- **[단계별 실행]** 모드로 한 단계씩 진행

{: .note }
> 막히면 LLM에게 오류 메시지를 그대로 붙여넣고 물어보세요.  
> 대부분의 오류는 LLM이 빠르게 해결 방법을 제시합니다.
