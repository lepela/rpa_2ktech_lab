# T2. 셀렉터 정제 가이드

웹 자동화에서 셀렉터가 깨질 때 빠르게 대처하는 방법을 정리합니다.

---

## PAD 자동캡처가 불안정한 이유

PAD가 자동으로 생성하는 셀렉터는 보통 이렇게 생겼습니다:

```css
:nth-child(3) > :nth-child(2) > span.value
```

이 셀렉터의 문제:
- "3번째 자식의 2번째 자식"이라는 **위치 기반** 선택
- 페이지에 광고 배너 하나 추가되면 순서가 바뀌어 **즉시 깨짐**
- 사이트 리뉴얼이나 A/B 테스트만으로도 **무효화**

---

## id vs class — 한 줄 설명

| 속성 | 설명 | 안정성 |
|------|------|--------|
| **id** | 페이지에서 단 하나뿐인 고유 이름 | ⭐⭐⭐ 최고 |
| **class** | 여러 요소가 공유하는 그룹 이름 | ⭐⭐ 양호 |
| **구조 경로** | 몇 번째 자식인지 위치로 표현 | ⭐ 불안정 |

**우선순위:** id → 의미있는 class → 구조 경로

---

## 단계별 셀렉터 정제 절차

```
① PAD 자동캡처로 일단 요소 잡기
② 실행 → 깨지는 것 확인
③ Chrome에서 Ctrl+Shift+C 누르기
④ 원하는 요소 클릭 → Elements 탭에서 코드 확인
⑤ 우클릭 → Copy → Copy selector
⑥ LLM에 붙여넣기:
   "이 셀렉터를 PAD에 최적화해줘. id/class 우선으로 단순화해줘."
⑦ 나온 셀렉터를 PAD에서 직접 입력 → 테스트
```

## 자주 쓰는 안정적인 셀렉터 패턴

### 패턴 1. id로 직접 선택

```css
#oilcon1
```
페이지에 하나뿐인 고유 id. 가장 안정적입니다.

### 패턴 2. id 안의 class

```css
#oilcon1 .oil_info
```
`oilcon1` id 안에서 `oil_info` class를 찾습니다.

### 패턴 3. id + 태그 + class 조합

```css
#oilcon1 .oil_info dl:first-child dd span.text-3
```
id를 기준점으로 잡고 의미있는 class로 좁혀갑니다.

### 패턴 4. 데이터 속성

```css
[data-id="fuel-price"]
```
개발자가 자동화를 위해 붙여둔 data 속성이 있으면 가장 안정적입니다.

---

## LLM 활용 셀렉터 최적화

```
이 CSS 셀렉터를 PAD 웹자동화에 최적화해줘.
id나 의미있는 class를 우선 사용하도록 단순화해줘:

[복사한 셀렉터 붙여넣기]
```

---

## 자주 발생하는 셀렉터 오류

| 오류 증상 | 원인 | 해결 |
|----------|------|------|
| "UI 요소를 찾을 수 없음" | 셀렉터가 구조 변경으로 무효화 | 셀렉터 다시 캡처 후 정제 |
| 엉뚱한 요소 클릭됨 | 같은 class가 여러 요소에 적용 | 더 구체적인 상위 id 추가 |
| 가끔만 작동 | 동적 로딩 요소 (AJAX) | 대기 액션 추가 |
| 로그인 후에만 안 됨 | 로그인 상태에서 구조가 다름 | 로그인 후 다시 캡처 |

{: .note }
> **동적 로딩 대기 방법:**  
> **[웹 자동화] > [웹 페이지의 UI 요소 대기]** 액션을 요소 클릭 전에 추가하세요.
